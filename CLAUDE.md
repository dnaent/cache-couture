## 1. Brand DNA & Ideology

### 1.1 The Triple Meaning
1. **Caché** *(French verb / adj)* — "to hide" / "hidden." Made so as to be difficult to see or find.
2. **Cachet** *(French noun)* — prestige, credit, status, importance. A stamp of approval.
3. **Cache** *(tech)* — high-speed storage; the hidden silicon/metal layer that powers digital systems.

The brand sits at the intersection of all three: **encrypted luxury, stealth status, digital utility.**

### 1.2 Core Ideology — "Encrypted Ownership"
- Stealth wealth, not overt logos.
- "Black Box" philosophy: minimalist exteriors, complex high-performance interiors.
- The garment is a *cold wallet* — a physical key to a digital identity.
- Uniform for the modern urban operator who lives fluently across physical and metaverse environments.

### 1.3 Brand Pillars
1. **Stealth** — anonymous silhouettes, monochrome palette, no loud branding.
2. **Utility** — borderless functional design that adapts to any urban environment.
3. **Encryption** — every garment carries hidden value (NFT counterpart, reflective detail, technical fabric).
4. **Cachet** — status earned through quiet competence rather than noise.

### 1.4 The Black Box — brand codename and identity
> Caché is not a fashion brand with a tech feature.
> Caché is a piece of consumer hardware that happens to be wearable.

This is the **Black Box** — Caché's internal brand codename and the load-bearing identity statement for the project. Every output (still, clip, caption, drop page) should make it legible without ever stating it. The Black Box is the throughline that ties §1.2 ("Black Box philosophy: minimalist exteriors, complex high-performance interiors") to the materiality cues in §4 and the Phygital Protocol in §5.

> *Not to be confused with* the "North Star" terminology used by the **Nova Ecosystem** project, which is entirely separate from Caché. Do not use "North Star" anywhere in this repo or in Caché-branded output.

## 2. Visual Identity System

### 2.1 Primary Logo
- **Wordmark:** `CACHÉ` — high-contrast Didone serif, capitalised, classical proportions.
- **Symbol Mark:** stylised microchip — square silhouette with pin/lead protrusions on all four sides, internal circuit/maze pattern. Direct visual reference to the technological meaning of *cache*.
- **Tagline Lockup:** `HIDDEN / APPROVAL` — small-caps, generously tracked. `APPROVAL` is rendered mirrored/upside-down. This mirrored typographic device is a brand signature and must be preserved wherever the tagline appears.

### 2.2 Couture Treatment
`CACHÉ` sits top of card, `COUTURE` bottom of card, matching serif weight and tracking. Used on definition cards, atelier labels, and high-end campaign covers.

### 2.3 Colour Palette

| Role | Hex | Use |
|---|---|---|
| Stealth Black | `#000000` | Primary garment colour, type, chip mark |
| Off Black | `#0A0A0A` | Photography backdrop low-key, secondary mass |
| Studio White | `#F4F2EE` | Marketing background (warm off-white) |
| Bone | `#FAFAF7` | Hi-key alt background |
| Gunmetal | `#2A2D31` | Hardware accents, reflective callouts |
| Iridescent / Reflective | *(rendered, not flat)* | Highlights on yarn, membranes, NFC label points |

All marketing imagery stays in this monochromatic register. No spot colour unless the asset is tied to a phygital/metaverse moment.

### 2.4 Typography
- **Display / wordmark:** Didone serif (Bodoni, Didot, or custom cut).
- **Subline / tagline:** tracked small-caps serif. Mirror-flip the second token where the `HIDDEN / APPROVAL` device is used.
- **Body / technical:** neutral grotesk (Inter, Söhne, Helvetica Now) for spec sheets, drop pages, metadata.

### 2.5 Photography & Art Direction
Reference: the three Cache Core lookbook stills (Light, Daily, Winter).
- Warm off-white seamless backdrop.
- Flat, even diffused front-fill lighting — no dramatic chiaroscuro.
- Models stand frontal, hands relaxed at sides, neutral expression.
- Full-body framing, feet visible.
- Multiple figures composed side-by-side as a single horizontal "lineup plate" is a brand signature.
- Casting: globally diverse — explicit visual signal of universality.
- Recurring accessory motifs: balaclavas, ski goggles, NY-style caps, technical gloves.

## 3. Collections

> **Naming:** Collections end in `-ware` (not `-wear`). The suffix is intentional brand vocabulary — it mirrors `hardware` / `software` / `firmware` and reinforces the Black Box (§1.4): Caché reads as consumer hardware that happens to be wearable. Use `Lightware`, `Dailyware`, `Darkware` in all copy, file names, LoRA triggers, and folder structures. The historical labels `Light`, `Daily`, `Winter`, and the `-wear` forms are deprecated.

### 3.1 Cache Core — Lightware
Sleek athletic minimalism. Track jackets, essential tees, tailored sweatpants, avant-garde cut-out crop tops, lightweight balaclavas, technical shorts. Year-round transitional layer.

### 3.2 Cache Core — Dailyware
Everyday cotton-led mid-layer and lightweight essentials. Heavy slightly oversized tees, essential cotton hoodies (slightly oversized), lightweight cotton half-zip pullovers, straight-leg essential cotton joggers, straight-leg cotton cargo pants, mid-thigh cotton shorts. The "everyday uniform" — cotton-forward, no synthetic mid-layers (those live in Darkware), no tactical/utility vests, no outerwear.

### 3.3 Cache Core — Darkware
Strict outerwear, heavy weights. Voluminous hooded puffer jackets, longline puffer coats, puffer gilets, heavy outerwear-grade fleece quarter-zips, heavy utility straight-leg cargo trousers (bottoms-as-outerwear edge case — padded, technical), ski goggles. Stealth survival. The Darkware register also showcases the brand's reflective hardware most clearly — gunmetal pulls, iridescent membranes, and reflective yarn read strongest against off-black backdrops in macro framing. **Exclusion gate:** no shorts, no lightweight tees, no cotton mid-layers, no lightweight bottoms — those belong in Dailyware.

## 4. Materiality & Construction Cues

Ratified 2026-06-29. Brand fabric strategy = **premium but accessible**: spend on the touchpoints
a customer feels and sees (weight, hardware, deep-black dye, trims, packaging) and economise on the
unseen substrate (synthetic fills, blends, linings). Fabric is mapped to the `-ware` tiers (§3) — when
generating product imagery or marketing visuals, use the materials for that tier; mix & match within a
tier, never put synthetics on a Dailyware cotton piece.

#### Tier material map (drives product specs AND ComfyUI/marketing generation)

| Tier | Garment | Material / weight |
|---|---|---|
| **Dailyware** (signature) | Tees | **100% heavyweight cotton**, 240–260 GSM combed ring-spun single jersey, garment-dyed black |
| | Tracksuits (hooded top + matching bottoms) | **100% heavyweight cotton**, 380–420 GSM brushed-back loopback / French terry |
| **Lightware** (entry / transitional) | Lightweight tees, crops | 180–210 GSM cotton or cotton/modal |
| | Track jackets, shorts | matte nylon-taslan / peached poly shell, or poly-elastane 4-way stretch knit |
| | Lightweight balaclava | fine-gauge merino-blend knit |
| **Darkware** (outerwear, top tier) | Puffer jackets / longline / gilets | matte recycled-nylon or ripstop shell + **recycled polyester wadding (~80–120 g)** (not down) |
| | Softshell / tech jackets | poly-spandex softshell bonded to micro-fleece |
| | Heavy fleece quarter-zips | poly polar-fleece or sherpa |
| | Utility cargos | cotton-nylon ripstop twill, DWR finish |
| **Accessories** | Ski masks (hero) | **100% merino wool**; beanies in merino/lambswool blend |
| | Bags (duffle / messenger) | coated Cordura / ripstop nylon, matte |
| | Socks | combed-cotton rib, woven logo |

**Matte finish is non-negotiable** on every fabric — peached / brushed / enzyme-washed, never glossy.

#### Perceived-value levers (cheap signals of luxury — emphasise in renders + specs)
- **Weight / GSM** — heft reads as quality; the single biggest signal.
- **Deep, non-fading black** — garment / pigment dye for rich blacks.
- **Gunmetal hardware** — matte-black YKK zips, engraved pullers, metal tags.
- **Trims** — woven chip-mark label (§4.1), drawcord tips, branded eyelets; small, off-centre, never large logos.
- **Subtle reflectivity** — black-out reflective yarn, iridescent seam membranes; activated only at glancing light angles, never high-vis.
- **Packaging + hidden inner labels** that double as the NFC/SKU touchpoint (§5).

### 4.1 The chip mark — canonical placement and finish

The Caché chip symbol (CLAUDE.md §2.1) is **woven directly into the garment fabric** — *not* applied as a separate stitched-on label patch. It appears at exactly four canonical positions:

1. Back of the hood, centred just below the hood crown seam.
2. Back-right trouser pocket.
3. Halfway up the right trouser leg, outer side.
4. Halfway up the left arm sleeve, outer side.

Two canonical finishes:

- **Subtle reflective yarn** (primary) — the embroidery reads as a **soft grey or light-grey tonal whisper** against the black weave under even diffused light. Only when raking light catches the yarn at a glancing angle does the chip briefly flash as a clean cool silver. Never stark high-contrast white-on-black.
- **Engraved gunmetal hardware plate** (sparing) — a small metal plate, used selectively in places where hardware reads better than embroidery.

The chip is **very small** and **off-centre**. It rewards close attention; it should not announce itself from across a room. This is the physical signature of the Black Box (§1.4) — the visible counterpart to the NFC chip embedded in the inner label (§5).

## 5. The Phygital Protocol

Every physical Caché garment carries:
- A unique **encrypted SKU**.
- An embedded **NFC chip** or QR-encoded tag in the inner label.
- A **1:1 tethered NFT** counterpart that auto-mints to the buyer's wallet at purchase.

Rules:
- **Tethered Ownership** — physical and digital cannot be separated. Selling the physical requires transferring the NFT; otherwise the item is "bricked" / de-authenticated.
- **Authentication** — the NFT is the certificate of authenticity. Counterfeits cannot mint.
- **Metaverse Skin** — the NFT renders as a wearable in supported metaverse platforms (Decentraland, The Sandbox, Roblox, Ready Player Me avatars).

Marketing must communicate this as the brand's flagship innovation, not a side feature.

## 6. Brand Voice & Language

- **Tone:** cold, confident, intelligent, slightly mysterious.
- **Register:** technical, minimal, precise. Short sentences. Short paragraphs.
- **Avoid:** hype, exclamation marks, slang, emoji, generic streetwear vocabulary ("drop," "fire," "heat," "dope").
- **Embrace:** *Protocol, Latency, Encrypted, Access, Utility, Concealed, Tethered, Hardware, Software, Mirror, Stealth, Operator, Cache, Chassis.*
- **Headline structure:** noun-led, often one or two kerned-out caps. `ENCRYPTED LUXURY.` `STEALTH PROTOCOL.` `TETHERED.`
- **Body copy:** treat as technical documentation written for a tasteful reader.

## 8. Recommended Model Stack

> Ratified 2026-05-19. The stack converged on **Qwen Image + WAN 2.2** rather than the
> originally-planned Flux + LTX/Hunyuan combination. Qwen Image is competitive with Flux
> for editorial photoreal on a 12 GB card, and WAN 2.2 14B I2V outperforms LTX/Hunyuan
> for cinematic motion at our VRAM ceiling. LTX is retained ONLY as a fast-draft tier.

### 8.1 Stills — Product / Editorial / Lookbook

| Purpose | Model | Notes |
|---|---|---|
| Hero product on figure | **Qwen Image fp8** (`qwen_image_fp8_e4m3fn.safetensors`) | Primary stills model. Pair with `qwen_2.5_vl_7b_fp8_scaled` text encoder + `qwen_image_vae`. |
| Editorial / fashion | **Qwen Image fp8** + Caché brand LoRAs | LoRAs to be trained on the lookbook stills + chip mark. |
| Fast iteration / mood | **Qwen Image** + `Qwen-Image-Lightning-4steps-V1.0` LoRA | 4-step Lightning inference for thumbnails and concepting. |

| Structural conditioning | **Qwen InstantX ControlNet-Union** | Pose / depth / canny in a single ControlNet. |
| Upscaling | **4x-UltraSharp** then **SUPIR** (tiled) | Final upscale to print-ready resolution. |

### 8.2 Video — Campaign, Social, Lookbook Motion

| Purpose | Model | Notes |
|---|---|---|
| Short cinematic clips (≤ 5s) | **WAN 2.2 14B I2V fp8** (`wan2.2_i2v_high_noise_14B_fp8_scaled` + `_low_noise_`) | Primary video model. Two-stage high/low noise sampler is canonical. |
| Image-to-video | **WAN 2.2 I2V** + `wan2.2_i2v_lightx2v_4steps_lora_v1` (high+low) | 4-step Lightning inference for batch social cuts. |
| Text-to-video | **WAN 2.2 ti2v 5B fp16** (`wan2.2_ti2v_5B_fp16.safetensors`) | Lighter t2v path when no keyframe exists. |
| All-in-one | **WAN 2.2 rapid mega AIO v4** (`wan2.2-rapid-mega-aio-v4.safetensors`) | Combined checkpoint when bouncing between modes inside one workflow. |
| Hyper-fast drafts | **LTX-Video 2B** | Storyboard previews before committing to WAN renders. Near real-time on 3080 Ti. |

### 11.3 LoRA Trigger Convention
The LoRA registry is organised into four asset classes plus a utility folder. See `ComfyUI/models/loras/cache/README.md` for the executable source and §16.4 for stacking precedence.

- **C1 — Faces** — `cache_face_m##.safetensors`, trigger `m##_face` (one per active cast slot, M01–M10; ten identities after the 2026-05-21 M03 reactivation)
- **C2 — Lightware** — `cache_garment_lightware.safetensors`, trigger `lightware_set`

Backdrop selection is a **prompt fragment**, not a LoRA. Use `on off-black seamless backdrop` (`#0A0A0A`) for stealth / reflective-hardware macro, or `on warm bone seamless backdrop` (`#FAFAF7`) for canonical lookbook register. The garment LoRAs are trained on both backdrops, so reflective physics are encoded in the garment LoRA itself.

`cache_brand` and `cache_core_*` are deprecated. Brand DNA is now carried by the face + garment LoRAs together; lineup composition is a compositing step, not a learned behaviour.

## 12. Content Production Workflows

### 12.1 Lookbook Lineup Plate
1. For each figure: render in Qwen Image stacking `cache_face_m##` (0.85–1.0) + `cache_garment_<collection>` (0.6–0.85). Use a single backdrop prompt across all figures (`on warm bone seamless backdrop` for canonical lookbook).
2. Lock backdrop and lighting via shared seed offsets (`seed`, `seed+1`, …).
3. Composite side-by-side at 1920 × 1080 (or 2560 × 1440 for hero). Lineup is a compositing step — no lineup LoRA.
4. Upscale with SUPIR (tiled).
5. Apply subtle film-grain matrix (≤ 2%) for cohesion.

### 12.2 Hero Product Still
1. Qwen Image base render at 1024 × 1536 with `cache_face_m##` + `cache_garment_<collection>`. Pick backdrop per intent — off-black for stealth/macro, bone for register-canonical.
2. Inpaint chip-mark embroidery at chest, sleeve, or back-pocket label using `cache_chip` LoRA.
3. Add reflective seam highlights via ControlNet line guidance.
4. Upscale → 4096 × 6144.

### 12.3 Campaign Video (5–8 s)
1. Generate hero keyframe in Qwen Image (locked seed) using the same face + garment LoRA stack as §12.2.
2. Feed into WAN 2.2 I2V with a motion prompt — e.g. *"slow dolly forward, model holds gaze, subtle wind on fabric."*
3. Render at 720p, 24 fps. Offload text encoder to CPU.
4. Upscale + RIFE-interpolate to 1080p 30/60 fps.
5. Layer MMAudio atmosphere, master in DaVinci Resolve.

### 12.4 Phygital Drop Card
1. Render product still as above.
2. Overlay QR code containing the encrypted SKU stub.
3. Place chip-mark in corner with `HIDDEN / APPROVAL` lockup.
4. Export 1080 × 1350 (IG portrait) and 1080 × 1920 (story).

## 13. Claude Code / Antigravity Behavioural Rules

When operating inside this project, Claude should:

1. **Default to Python + ComfyUI orchestration.** Reusable scripts in `scripts/`. Avoid one-off shell pipelines.
2. **Talk to ComfyUI via its WebSocket / REST API**, not by automating the browser UI. Use `comfy_client.py` as the shared client.
3. **Treat workflow JSONs as source code.** Lint, diff, and version them. Never edit them inside ComfyUI without exporting back to the repo.
4. **Respect the brand voice in every generated caption, alt-text, and metadata field.** Cold, precise, minimal.
5. **Never invent palette colours, taglines, or product names** outside this document. If something new is needed, propose it and update `CLAUDE.md` *before* using it.
6. **VRAM awareness:** before queuing any new model, confirm it fits 12 GB with offload. Default to fp8 / GGUF Q4–Q5.
7. **Frontend deliverables:** Firebase Hosting for static drop pages; Canva Code for fast social one-pagers; React + Tailwind for the main marketing site.
8. **Backend deliverables:** Python (FastAPI) for APIs (SKU → NFT lookup, mint webhook handlers). Firebase Functions for lightweight glue.
9. **Always log generations.** Every output written to `outputs/` ships with a sidecar `.json` containing model, LoRAs, seed, prompt, negative, workflow hash, and timestamp.
10. **Ask before destructive operations** on `brand/` or `workflows/`.

## 16. Consistent Cast Convention

### 16.1 Why a Fixed Cast
The Caché brand voice depends on the same operators recurring across drops, lookbooks, and campaign motion. Random per-render faces dilute the "uniform" signal. The brand therefore maintains a fixed cast of **M01..M10** — ten identity slots, each backed by a dedicated face LoRA.

### 16.2 Slot Layout
- Slots are numbered `M01` through `M10`. The `M` prefix is a brand-internal token; do not surface it in customer-facing copy.
- Each slot owns a directory under `brand/models/M##/` containing a `profile.md`, a `source/portrait.jpg`, a `training_set/`, and a `captions/` folder.
- The roster table at `brand/models/INDEX.md` is the index of truth — update it when slots are assigned, retrained, or retired.
- New cast members get the next free `M##` slot; do not renumber existing slots, as their LoRAs are referenced by token elsewhere.

### 16.3 LoRA Naming
- Face LoRA file: `ComfyUI/models/loras/cache/c1_faces/cache_face_m##.safetensors`
- Trigger token (in prompts): `m##_face` — e.g. `m07_face`
- Training rank: **16–32** (rank 16 default; rank 32 only if the face is structurally unusual)
- Training-set size: **20 images** per face — varied angles, lighting matches the warm off-white seamless reference

### 16.4 Stacking Order
Reproduced from `ComfyUI/models/loras/cache/README.md` (which remains the executable source):
1. Base model (Qwen Image)
2. `cache_face_m##`              (0.85 – 1.0; never below 0.75 or identity drifts)
3. `cache_garment_<collection>`  (0.6 – 0.85 — Lightware / Dailyware / Darkware)
4. `cache_chip` (optional)       (0.5 – 0.7, only on assets requiring chip-mark insertion)
5. Lightning step LoRA           (e.g. `Qwen-Image-Lightning-4steps`)

Backdrop is a **prompt fragment**, not a LoRA — `on off-black seamless backdrop` (`#0A0A0A`) for stealth/macro, `on warm bone seamless backdrop` (`#FAFAF7`) for canonical lookbook register. Garment LoRAs are trained on both backdrops so reflective physics travel with the garment.

### 16.5 Consistency Pipeline
The full three-tier pipeline (Qwen Image Edit → bootstrap training set → per-model face LoRA → garment-LoRA stacking + compositing) is documented in `.claude/consistency_pipeline.md`. Always consult that file before training a new face or proposing a workflow change that touches identity lock.
